5 Funnel dashboard — built & verified

The per-source clicks ÷ registrations → conversion % view Tal requested is now wired into getPromoReporting (both base44/…/entry.ts and supabase/…/index.ts). The UI (HumondialReporting.jsx → "הרשמות לפי מקור") already rendered the columns, so they light up automatically.

  • Groups registrations by origin_source, joins Short.io humanClicks via short_io_slug over the selected date window, computes conversion %.
  • Short.io key read from the Base44 runtime env var SHORT_IO_API_KEY — never in the repo. Fully non-fatal: if the key/API is unavailable, clicks show blank and the rest of the report still renders.
  • Includes active sources with zero registrations but non-zero clicks → surfaces funnel leaks.
  • Per-branch click granularity intentionally omitted (in-venue QRs are generic).
