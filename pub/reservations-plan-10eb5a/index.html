Implementation Plan

Late-Match In-App Reservations

Goal: Replace the Tabit reservation handoff with an in-app flow for matches after 23:00 (Asia/Jerusalem), plus a per-branch manager view (PWA) and a chain-wide HQ view (backoffice) of who's coming to each game.

Architecture: New promo_reservations table + promo_matches.reservation_mode. Five Base44 functions (create/read for players, read/mutate for managers/HQ). PWA gets a reservation form + gated /manager route; backoffice gets a CMS toggle + read-only HQ screen. Points reuse the existing pending-table_booking mechanic.

Supabase Postgres 17Deno · Base44 fns React + ViteVitestTailwind
